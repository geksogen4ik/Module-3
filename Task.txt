1. Есть библиотека, в которой находятся разные типы книг.
2. Каждый тип характеризуется:
* Название
* Длительность в днях, на которую выдается книга.

3. С клавиатуры осуществляется ввод книг, для заполнения библиотеки. 
Ситуацию с базовым типом я бы советовал решить при помощи панели выбора, например

- Книга(1)
- Периодическое издание (2)
- Многотомник (3)
- ...

4. По данному набору происходит выдача книг читателям, а также в случайном порядке - возврат книг. Каждый из процессов имеет внутри собственный шаг (день), которые совпадают.
5. Учитывать просрочку книг по сроку из пукнта 2, результаты просрочки сохранять в отдельный отчет
6. Иметь возможность показать книги в наличии и книги на руках.

Бонус - отдельный поток по генерации новых тасков
